# C++ JSONL 筛选流程记录

本文档记录从 `full.jsonl` 到最终 C++ 结果文件的完整筛选流程。

---

## 步骤 1：按 C++ 项目名筛选

- 输入文件：`full.jsonl`（249830）
- 脚本：`extract_cpp_projects.py`
- 逻辑：读取每条 JSON，检查 `repo` 是否属于 5 个 C++ 项目名单（arrow, brpc, mesos, mxnet, thrift），命中则保留。
- 输出文件：`full_cpp_projects.jsonl`（16468）

---

## 步骤 2：整合筛选与打标（简化版流水线）

为了提高准确率并简化流程，我们采用了“以 Diff 类名为锚点，在 Message 中进行目标搜索”的策略。

- 输入文件：`full_cpp_projects.jsonl`（16468）
- 脚本：`filter_cpp_dataset.py`
- 逻辑：
  1. **解析 Diff**：从 `.cpp`, `.h` 等源文件中提取 `class`, `struct`, `namespace` 涉及的类名。
  2. **目标匹配**：将提取出的类名作为关键词，在 commit message 中进行匹配。
     - **动态大小写敏感**：如果 Diff 涉及的类名列表中不存在仅大小写不同的冲突项，则进行不区分大小写的匹配；若存在冲突，则对冲突项进行区分大小写的精确匹配。
     - 支持全称和简称，并使用单词边界匹配。
  3. **均衡筛选**：
     - 涉及类总数 $N \ge 2$。
     - 被提到的类（Positive）数量 $P \ge 1$。
     - 未被提到的类（Negative）数量 $M \ge 1$。
- 输出文件：`full_cpp_balanced.jsonl`（1545）

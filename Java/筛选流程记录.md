# Java JSONL 筛选流程记录

本文档记录从 `full.jsonl` 到最终 Java 结果文件的完整筛选流程。

---

## 步骤 1：按 Java 项目名筛选

- 输入文件：`full.jsonl`（249830）
- 脚本：`extract_java_projects.py`
- 逻辑：读取每条 JSON，检查 `repo` 是否属于 34 个 Java 项目名单，命中则保留。
- 输出文件：`full_java_projects.jsonl`（172277）

---

## 步骤 2：整合筛选与打标（简化版流水线）

采用与 C++ 和 Python 版本一致的“以 Diff 类名为锚点，在 Message 中进行目标搜索”的策略。

- 输入文件：`full_java_projects.jsonl`（172277）
- 脚本：`filter_java_dataset.py`
- 逻辑：
  1. **解析 Diff**：从 `.java` 源文件中提取 `class`, `interface`, `enum`, `record` 涉及的类名。
  2. **目标匹配**：将提取出的类名作为关键词，在 commit message 中进行匹配。
     - **动态大小写敏感**：若 Diff 类名列表中无大小写冲突，则不区分大小写匹配；若有冲突，则对冲突项区分大小写匹配。
     - 使用单词边界匹配。
  3. **均衡筛选**：
     - 涉及类总数 N ≥ 2。
     - 被提到的类（Positive）数量 P ≥ 1。
     - 未被提到的类（Negative）数量 M ≥ 1。
- 输出文件：`full_java_balanced.jsonl`（18726）

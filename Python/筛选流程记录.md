# Python JSONL 筛选流程记录

本文档记录从 `full.jsonl` 到最终 Python 结果文件的完整筛选流程。

---

## 步骤 1：按 Python 项目名筛选

- 输入文件：`full.jsonl`（249830）
- 脚本：`extract_python_projects.py`
- 逻辑：读取每条 JSON，检查 `repo` 是否属于 2 个 Python 项目名单（airflow, tvm），命中则保留。
- 输出文件：`full_python_projects.jsonl`（20081）

---

## 步骤 2：整合筛选与打标（简化版流水线）

采用“以 Diff 符号名为锚点，在 Message 中进行目标搜索”的策略。

- 输入文件：`full_python_projects.jsonl`（20081）
- 脚本：`filter_python_dataset.py`
- 逻辑：
  1. **解析 Diff**：从 `.py` 源文件中提取 `class`, `def` 涉及的符号名。
  2. **目标匹配**：将提取出的符号名作为关键词，在 commit message 中进行匹配。
     - **动态大小写敏感**：若 Diff 符号列表中无大小写冲突，则不区分大小写匹配；若有冲突，则对冲突项区分大小写匹配。
     - 使用单词边界匹配。
  3. **均衡筛选**：
     - 涉及符号总数 $N \ge 2$。
     - 被提到的符号（Positive）数量 $P \ge 1$。
     - 未被提到的符号（Negative）数量 $M \ge 1$。
- 输出文件：`full_python_balanced.jsonl`（5037）
